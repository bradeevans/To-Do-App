## MEDIUM Severity Gaps

### Auto-Backup to Local Folder

- **What it does:** Automatically backs up all tasks to a user-selected folder on the local filesystem every 4 hours. Uses the browser's File System Access API to write a dated JSON file (`todo-backup-YYYY-MM-DD.json`). Stores the folder handle in IndexedDB so the selected folder persists across page reloads. A `BackupStatus` component in the bottom-right corner shows the last backup time and lets the user change the folder.
- **Files:**
  - [src/hooks/useAutoBackup.js](src/hooks/useAutoBackup.js)
  - [src/components/BackupStatus.jsx](src/components/BackupStatus.jsx)
  - [src/App.jsx](src/App.jsx) (lines 13–14, 38, 149–153)
- **Why it matters:** This is the only feature that writes data outside the browser (to the local filesystem) and uses a secondary storage system (IndexedDB). It's a meaningful capability that `SPEC.md` has no record of.
- **Recommendation:** Add an "Auto-Backup" section to `SPEC.md` documenting the trigger interval, file format, storage mechanism, and browser support requirement (File System Access API).

## LOW Severity Gaps

- **Within-column reordering** — Tasks can be dragged to reorder within the same column. `SPEC.md` mentions drag between columns but not same-column reordering. [src/App.jsx](src/App.jsx) lines 110–119.
- **Column empty state** — Columns display "Drop tasks here" placeholder when empty. Not mentioned in `SPEC.md`. [src/components/Column.jsx](src/components/Column.jsx) lines 36–38.
- **Done column strikethrough** — Tasks in the Done column display with a strikethrough title. Not mentioned in `SPEC.md`. [src/index.css](src/index.css) lines 242–248.
- **Drag activation distance** — A 6px pointer movement is required before drag begins, to prevent accidental drags on click. [src/App.jsx](src/App.jsx) lines 44–48.
- **Description editing keyboard behavior** — `SPEC.md` says "changes saved on blur or Enter" for all editing. In the code, description editing saves on blur but **not** on Enter (only Escape to cancel). Title editing does support Enter. [src/components/TaskCard.jsx](src/components/TaskCard.jsx) lines 84–96.
